---

## 1. 文档目的

本文档将产品能力拆为 **「当前重点打磨」** 与 **「后续可扩展」** 两大部分，便于：

1. **现阶段**：集中资源验证「从灵感到满意成稿」的核心价值，不分散到发布自动化。
2. **后续阶段**：内容质量稳定后，按模块接入发布校验、图片规格、一键发布等能力，且与现有数据模型、创作室 UI 兼容。
3. **对标参考**：相对 [Postudio](https://postudio.ai/zh) 等发布型工具，明确差异化边界与可借鉴能力。

**原则（与 PRD §1.7、§1.8 一致）**：

- 先 **创作质量**，后 **发布效率**。
- 一键/自动化发布 **不在当前阶段实现**，但在本文档中预留接口与数据扩展点。
- 不做自媒体矩阵、爆款套路生成器。

---

## 2. 阶段总览

```text
Phase 1 · 创作质量打磨（当前重点）     ← 你现在在这里
    ↓ 成稿满意度 ≥ 目标后再进入
Phase 2 · 发布前体验增强
    ↓ 校验 + 图片规格 + 发布记录完善
Phase 3 · 自动化发布与进度追踪         ← 内容打磨完成后再做
    ↓
Phase 4 · 运营复盘与平台扩展
```

| 阶段 | 主题 | 目标一句话 | 建议触发条件 |
| --- | --- | --- | --- |
| **Phase 1** | 创作质量打磨 | 用户觉得「像自己写的、可直接复制发布」 | MVP 已上线，进入内部试用 |
| **Phase 2** | 发布前体验增强 | 发布前知道「能不能发、规格对不对」 | Phase 1 核心指标达标 |
| **Phase 3** | 自动化发布 | 成稿后一键分发，可见进度与结果 | Phase 2 校验稳定 + 用户明确需要 |
| **Phase 4** | 复盘与扩展 | 长期创作资产与多平台增长 | Phase 3 或手动发布流程跑通 |

> **本阶段不做**：平台账号绑定、浏览器自动化发布、发布进度条、桌面客户端。

## 5. Phase 3 — 自动化发布与进度追踪

> **实施时机**：Phase 1 创作质量达标，且 Phase 2 校验与发布记录跑通后。  
> **用户原话**：「内容打磨完成后再实现一键/自动化发布 + 发布进度」。

### 5.1 架构原则

```text
ContentProject（status = ready）
    → PublishValidator（复用 Phase 2 规则）
    → PublishJob（队列 / 单任务）
    → PlatformPublisher（策略模式，每平台一个 Adapter）
    → PublishProgress（SSE / WebSocket 推送）
    → PublishRecord（结果 + 截图 + 链接）
```

- **策略模式**：`WechatPublisher` / `XiaohongshuPublisher` / `DouyinPublisher` 独立实现，便于增删平台。
- **失败隔离**：单平台失败不影响其他平台，支持 **单条重试**。
- **默认推荐**：浏览器自动化（Playwright）+ **本地凭证**；官方 API 作为可选 Adapter。

| 维护成本 | 发布模块独立 repo / 包，与创作室解耦 |

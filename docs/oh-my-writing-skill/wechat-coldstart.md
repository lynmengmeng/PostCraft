# 公众号冷启动内容规则

新号流量主要来自 **搜索 + 私域转发**，而非系统推荐。以下规则帮助文章更容易被目标读者搜索、点开、读完、转发。

## 1. 标题：搜一搜能搜到，推荐流想点开

**类比理解：** 标题像店招——路人（推荐流）只有 1 秒决定是否进门；搜一搜用户像按图索骥，招牌上必须写清「卖什么、给谁、解决啥」。

### 高打开率三要素（每条标题尽量覆盖 2/3 以上）

| 要素 | 作用 | 写法提示 |
| --- | --- | --- |
| **社交币** | 读者愿意转发/收藏，显得有见识 | 纠误区、反常识、「别再…」「很多人不知道…」、可转给家人 |
| **悬念** | 留好奇缺口，促点击 | 问句、数字未揭晓、转折「却/原来/竟然」 |
| **利益点** | 读者立刻知道「看了能得到什么」 | 人群 + 痛点 + 可执行结果/判断 |

**推荐模板：数字 + 结果 + 悬念**

- 农村子女，老人总说「没事」？先查这 3 个信号
- 普通学生怎么逆袭？学姐说的「先做起来」，其实是最有效的方法
- AI 会让程序员失业吗？选专业前这 3 个判断

### 从普通到高打开（对比示例）

| 普通（难搜、难点） | 优化后 |
| --- | --- |
| 如何提高学习能力 | 普通学生怎么逆袭？那个学姐反复说的「先做起来」，其实是最有效的方法 |
| AI 来了，程序员开始失业，然后呢？ | AI 会让程序员失业吗？给学生选专业前的 3 个判断 |
| 清华学霸的暑假怎么过 | 清华学霸暑假怎么安排？每天抓住这 3 块时间，普通学生也能用 |
| 那个学姐反复说「先做起来」，后来她们都火了 | （同上第二行：保留故事感，但补上人群 + 问句 + 利益点） |

**避免过泛标题**（用户不会搜、也不会点）：
- 「如何提高 X 成绩 / 能力」
- 「X 全面指南 / 入门攻略」
- 「关于 X 的一些思考」

**优先使用以下公式**（从选题/灵感提取具体场景、人群、动作词）：

| 公式 | 示例 |
| --- | --- |
| 人群 + 痛点 + 结果 | 农村子女，老人总说「没事」却反复住院？先查这 3 件事 |
| 问题 + 解决方案 | 三无保健品包装像正规药？识别方法其实在这几个细节 |
| 误区 + 正确做法 | 别再只看价格了，买这类日用品最容易踩的坑在这 |
| 对比 + 选择建议 | 社区医院和县城三甲，什么情况该直接去后者？ |
| 数字 + 结果 + 悬念 | 家长最容易踩的 3 个教育误区，第三个最常见 |

**标题硬性要求：**
- 不超过 64 字（过长在信息流里会被截断，点击意愿下降）
- 含可搜索关键词：具体场景、人群、动作词
- 不用震惊体、哗众取宠词（「必看」「震惊」「疯传」「不看后悔」等）
- 不用纯文艺、纯感悟式标题（有情绪可以，但必须带利益点或悬念）

## 2. 开头：前 3 段必须抓人（约 80–120 字）

**禁止的开头：**
- 「随着……越来越……」类宏观背景铺垫
- 品牌故事、理念介绍、行业综述
- 「全文 X 字，阅读约需 X 分钟」作为开头元素
- 「最近某现象很火，朋友圈里到处都在讨论」

**推荐结构（按顺序写 3 段）：**
1. **具体痛点场景**：读者一看就知道「这和我有关」
2. **反常识判断**：打破读者默认假设（「问题可能不是 X，而是 Y」）
3. **本文承诺**：这篇文章讲清楚什么、解决什么

**示例：**

```
很多农村家庭不是不关心老人，而是老人嘴上说「没事」，子女根本不知道该怎么判断。

最常见的问题不是不够孝顺，而是分不清「可以观察」和「必须就医」的信号。

这篇文章讲清楚：老人反复说没事时，子女应该先问哪几个问题、看哪几个细节。
```

## 3. 正文：每篇只解决一个问题

**硬性约束：**
- 从灵感/选题提取 **1 个核心问题**，全文只回答这一个问题
- 禁止一篇覆盖多个独立话题（如同时讲现象背景、多个案例方向、多个不相关建议）
- 正文 3–5 个小节，每节对应一个可执行步骤或判断点
- 每段 2–4 行，适合手机阅读
- 每 300–500 字给 1 个具体例子或场景

**推荐结构：**

```markdown
## 核心问题

[一句话说明本文只回答什么]

## 一、[第一个判断点/步骤]

...

## 二、[第二个判断点/步骤]

...

## 你可以先做这一步

[1 个明确、可执行的行动建议]

[可选：若 topic_meta.series 有值，预告下篇系列主题]
```

**排版预设：** 步骤清单、干货罗列类优先用 `checklist` layout_preset。

## 4. 结尾

- 给 **1 个明确行动**（适合收藏、可以转给相关的人）
- 若元信息中有 `series` 系列名，可预告下篇；无则不必强行预告
- 不用空洞的「希望对你有帮助」

## 5. 封面文案

输出 JSON 时额外提供：

- **cover_headline**：一句痛点问句，≤14 字
- **cover_subheadline**：具体结果或方法，≤18 字

**示例：**
- headline：老人总说没事？
- subheadline：子女先查这 3 个信号

**视觉 prompt 要求：** 简洁、横版 2.35:1、主体居中便于方形裁切、少装饰、不要明显 AI 感。

## 6. 摘要（summary）

- 120 字以内，用于朋友圈分享
- 写痛点 + 价值，不要写背景综述
